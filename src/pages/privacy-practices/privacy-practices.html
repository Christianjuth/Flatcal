Privacy Practices

I am a strong believer in privacy and protecting anonymity online. Data is only collected with the intension of improving FlatCal. Additionally, I have done my best to remove any information that could be considered personal identifiable. I do not collect names, emails, or exact location's of any users! I do not even have access to that information. I have attempted to scramble IP addresses, and I assure you I will continue to do my best to protect the anonymity of FlatCal users. FlatCal is opensource, and you can see all the sourcecode here. FlatCal's sourcecode falls under The MIT license.

Examples of Data We Collect:

  • User ID - Randomly generated to assosiate crashes with users (ex. 65460cdc-0a83-d9fa-e7b3-2974a79c3d39).
  • Pages Visited. We track which pages in FlatCal are used to improve the product.
  • Crash Reports. Software isn't perfect. We log error info when we mess up to help debug FlatCal.
  • Features Used. We track which FlatCal features are popular in order to improve the product.
  • Browser Version. This helps us see which browsers we need to test FlatCal on the most.
  • Operating System. FlatCal adapts to meet OS needs. This info helps debug crashes and improve the product.

Services We Use:

  • Google Analytics - analytics
  • Sentury.io - crash reporting

User ID is used in place of IP addresses because it is both more anonomous and more consistent. Analytics and crash reports help me diagnose issues with FlatCal and figure out which features are worth focusing on. There is currently no way to opt out of this anonymous data collection. FlatCal is a side project, it is free to use, and this data helps me deliever the best expierence possible to users. I hope you understand, and thank you for using FlatCal.
